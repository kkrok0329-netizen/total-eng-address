## 브라우저 검사 방식과 제한

실제 Chromium 엔진으로 모바일 레이아웃, 검색, 상세보기, 즐겨찾기와 데이터 렌더링을 검사했습니다. 작업 환경의 브라우저 관리 정책이 로컬 HTTP 주소 직접 탐색을 `ERR_BLOCKED_BY_ADMINISTRATOR`로 차단하여, 테스트 문서를 Chromium에 직접 주입하고 동일한 CSS·JavaScript·JSON·이미지를 연결하는 방식으로 UI를 검증했습니다.

따라서 GitHub Pages에 올린 뒤 실제 기기에서 다음 두 항목은 마지막으로 확인해야 합니다.

1. Android Chrome에서 `앱 설치` 또는 `홈 화면에 추가`가 표시되는지
2. iPhone Safari에서 `공유 → 홈 화면에 추가` 후 독립 앱 화면으로 열리는지
